# CS305SoftwareSecurity
Briefly summarize your client, Artemis Financial, and its software requirements. Who was the client? What issue did the company want you to address?
Artemis Financial was the client, and the company wanted its software reviewed for security weaknesses. The main issue was identifying vulnerabilities in its web application and improving protections for sensitive financial data during transmission and storage.

What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely? What value does software security add to a company’s overall well-being?
I did well at examining the project for software security vulnerabilities and identifying areas that could put the application at risk. Secure coding is important because it helps protect confidential data, prevents attacks, and reduces the chance of downtime, financial loss, or damage to a company’s reputation. Strong software security adds value by improving trust, reliability, and compliance.

Which part of the vulnerability assessment was challenging or helpful to you?
One challenging part of the vulnerability assessment was understanding which findings were the most serious and which mitigation steps would be most effective. At the same time, this was also helpful because it showed how important it is to prioritize risks instead of treating every issue the same way.

How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?
I increased layers of security by using stronger encryption, reviewing dependencies, and applying secure coding practices to reduce risk. In the future, I would continue using vulnerability scans, dependency-checking tools, manual code reviews, and secure coding standards to assess weaknesses and choose the best mitigation techniques.

How did you make certain the code and software application were functional and secure? After refactoring the code, how did you check to see whether you introduced new vulnerabilities?
I made certain the code and application were functional and secure by testing the program after refactoring and rerunning security checks. After making changes, I reviewed the results again to confirm the application still worked correctly and that no new vulnerabilities were introduced.

What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks?
Some helpful resources and tools I used were OWASP Dependency-Check, secure coding practices, Java/Spring security concepts, code reviews, and testing after changes were made. These are useful because they help identify risks early and support safer development.

Employers sometimes ask for examples of work that you have successfully completed to show your skills, knowledge, and experience. What might you show future employers from this assignment?
For future employers, I could show this assignment as an example of my ability to perform a vulnerability assessment, identify software risks, apply mitigation strategies, refactor code securely, and verify both functionality and security after making improvements. It demonstrates problem-solving, attention to detail, and knowledge of secure software development.
